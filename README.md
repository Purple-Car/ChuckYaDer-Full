# ChuckYaDer-Full

## Propósito

Este é um projeto acadêmico de Portifólio como Trabalho de Conclusão de Curso em Engenharia de Software para a universidade Católica de Santa Catarina.

## Descrição

Esta versão de 'ChuckYaDer' trata-se de um jogo eletrônico co-operativo de plataforma focado em solução de puzzles. O projeto em si é uma versão completamente refeita e expandida do original, dessa vez rodando na Godot Engine.

## Controles

### Menus

Os menus do jogo são controlados majoritáriamente pelo Mouse, a única exceção sendo a seleção de cores em que cada jogador controla seu cursor, e o botão ESC que pausa o jogo durante gameplay.

### Personagens

O jogo conta com 2 jogadores que podem ser controlados através do mesmo dispositivo, e os controles são simples, contando apenas com movimentação e um botão de ação. Segue abaixo os controles padrões do jogo, que podem ser também conferidos e alterados no menu "Opções":

| Command | Player 1 | Player 2 |
| ------------- | ------------- | ------------- |
| Cima | W | ↑ |
| Baixo | S | ↓ |
| Esquerda | A | ← |
| Direita | D | → |
| Ação | shift | ctrl |

## Tecnologia e Execução

O projeto foi desenvolvido na Godot Engine, o que significa que ele é composto quase inteiramente de gdscript, uma linguagem própria. Testes e deploy são realizados através do próprio Godot, e sendo um jogo offline ele é distribuído de forma contida.

O conteúdo presente em uma cópia são o executável e um arquivo .pck (padrão do Godot) contendo todos os recursos do jogo. Para rodar basta garantir que os dois estão na mesma pasta e rodar o executável.
